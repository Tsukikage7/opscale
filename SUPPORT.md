# Support

Use GitHub issues for reproducible bugs and focused feature requests.

Before opening an issue:

- Confirm the package version or commit SHA.
- Redact database credentials, tokens, customer data, and production DSNs.
- Include the database dialect and a minimal schema or command that reproduces the issue.
- Run `pnpm run verify` for local development problems.

Security vulnerabilities should follow [SECURITY.md](./SECURITY.md), not public issues.
